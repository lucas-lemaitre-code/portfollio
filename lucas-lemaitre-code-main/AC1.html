Compétence 1 : Administrer


AC11.01 | Lois fondamentales de l’électricité


Ce que j'ai fait : Étude des bases de l'électricité (Tension, Courant, Puissance) appliquée aux équipements.

Pourquoi je l'ai fait : Pour comprendre les besoins énergétiques des baies de brassage et éviter les surcharges.

Comment je l'ai fait : Cours de R104 et mesures en TP.

Mes difficultés : Passer de la théorie mathématique à la pratique sur le matériel.

Ce que j'en ai appris : L'importance de la sécurité électrique en salle serveur.

Ce que je ferais autrement : Faire plus de schémas de câblage électrique avant l'installation physique.

AC11.02 | Architecture et fondements (IP, Internet)


Ce que j'ai fait : Conception d'un plan d'adressage IPv4 pour les 4 VLANs de la SAÉ 1.02.

Pourquoi : Assurer la communication entre les hôtes tout en segmentant les services.

Comment je l'ai fait: Calcul de masques de sous-réseau et annotation sur draw.io.

Difficultés : Comprendre la logique binaire des masques au début.

Appris : La hiérarchie du modèle OSI (Couche 3).

Autrement : Utiliser un tableur pour valider mes plages d'adresses plus rapidement.

AC11.03 | Configurer les fonctions de base du LAN


Ce que j'ai fait : Configuration de switchs Cisco 2960 et routeurs 800 dans la SAÉ 1.02.

Pourquoi : Créer les VLANs (ADMIN, PROD, etc.) demandés par le cahier des charges.

Comment je l'ai fait: Lignes de commande Cisco (CLI) et mode Trunk (802.1Q).

Difficultés : Les erreurs de syntaxe dans les commandes IOS.

Appris : La différence entre le mode "access" et "trunk".

Autrement : Préparer mes scripts de configuration dans un bloc-notes avant de les injecter.

AC11.04 | Maîtriser les systèmes d’exploitation (OS)


Ce que j'ai fait : Installation d'un serveur Web Apache sur Linux (Debian) dans la SAÉ 1.02.

Pourquoi : Héberger la page statique présentant l'entreprise.

Comment je l'ai fait: Utilisation du terminal Linux et gestion des fichiers de config dans /etc/.

Difficultés : Gestion des droits d'accès aux fichiers (chmod).

Appris : L'administration de base d'un système Linux sans interface graphique.

Autrement : Automatiser l'installation du serveur via un script Bash.

AC11.05 | Identifier les dysfonctionnements


Ce que j'ai fait : Tests de connectivité (Ping, Traceroute) et capture de trames (Wireshark).

Pourquoi : Vérifier que les VLANs communiquent correctement ou sont bien isolés.

Comment je l'ai fait: Protocole de tests rigoureux avec captures d'écran des résultats.

Difficultés : Analyser les trames Wireshark complexes.

Appris : La méthode de dépannage "bas en haut" (du physique vers le logiciel).

Autrement : Noter systématiquement chaque erreur rencontrée et sa solution.

AC11.06 | Installer un poste client


Ce que j'ai fait : Configuration de machines virtuelles Windows et Linux pour simuler les employés.

Pourquoi : Valider le fonctionnement du serveur DHCP installé sur le routeur.

Comment je l'ai fait: Configuration des interfaces réseau en mode DHCP.

Difficultés : Conflits d'adresses IP lors des premiers essais.
Appris : Le cycle de vie d'une requête DHCP (DORA).

Autrement : Vérifier les baux DHCP directement sur le routeur pour monitorer les clients.
